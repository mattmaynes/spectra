Spectra now uses the shared Trellis plugin-release pipeline for versioning and releases.

## What changed
Spectra adopted the Trellis `plugin-release` template, retiring its hand-rolled release mechanics.
A root `VERSION` file is now the single source of truth, mirrored into all seven manifests by
`scripts/bump-version.sh` (with a `--check` CI drift guard), and merging a bump auto-tags and
publishes the release - which then refreshes this "What's new" headline. Spectra and Trellis run
the identical pipeline, owned and kept current by `/trellis-update`.

This release is the first cut through that pipeline.
